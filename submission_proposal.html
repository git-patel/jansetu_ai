The JanSetu Paradigm

Instead of requiring digital literacy, JanSetu AI enables citizens to report needs via localized voice recordings or photos. The platform automatically translates regional dialects, estimates resource costs, deduplicates reports geospatially, and ranks requests objectively on a 0-100 severity scale.

2. Problem Statement

Constituency planning and budget allocations (such as the MPLADS fund in India) are slow and lack transparency:

Unstructured Citizen Feedback

Citizens struggle with bureaucratic jargon, leading to poorly routed, text-heavy tickets in local dialects that stall in database queues.

The Duplicate Submission Trap

Multiple residents independently report the same local issue (e.g., a pothole near a school), wasting administrative resources on duplicate inspection visits.

Opaque Budget Allocations

Public representatives (MPs) manage annual budgets up to ₹5 Crore with limited data. Funding decisions are often made without clear metrics, leading to inefficient spending.

Disconnected Stakeholder Feedback Loops

No single tool connects citizen reports to budget approvals, contractor assignments, and geotagged project completion verification.

3. Our Solution

JanSetu AI connects the entire governance lifecycle into a single digital ecosystem:

Citizen Ingestion

Constituents submit local needs via photo or voice. AI translates dialects and categorizes the issue automatically.

MP Command Board

Provides representatives with AI-sorted priority lists and a one-click budget sanctioning workflow.

Admin Oversight

Allows state administrators to monitor department performance, release funds in escrow, and audit project metrics.

4. System Architecture

JanSetu AI is built on **Clean Architecture** and SOLID design principles, dividing the codebase into modular layers to isolate the user interface from underlying cloud repositories:

Dynamic Data Source Switcher: The system includes a global `ServiceLocator` that checks configurations at startup. It initializes either offline mock databases (`MockAiProvider`) or active cloud connections (`GeminiAiProvider`) without requiring changes to the user interface widgets.

6. Complete User Journey

This timeline illustrates how a citizen's report moves through verification and funding approval:

Stage 1: Issue Reported

A resident uploads a photo and describes a local water leak in Gujarati. The voice input is automatically translated to English, and the issue is routed to the Water & Sanitation department.

Stage 2: Spatial Deduplication

The AI checks local coordinates. If a similar issue is already open nearby, it merges the reports to prevent duplicates and alerts the citizen.

Stage 3: AI Priority Scoring

The system calculates a 10.0 to 99.5 priority score based on severity, upvotes, and population impact, ranking the report in the queue.

Stage 4: MP Budget Approval

The MP reviews the prioritized queue on their dashboard and approves the project with one click, deducting funds from their MPLADS budget.

Stage 5: Milestone Progress & Payouts

Administrators assign PWD engineers to lead construction. Budgets are released in escrow tranches as the contractor completes verified phases.

10. Artificial Intelligence

The platform's AI functions are managed by a centralized, decoupled **Gemini AI Engine** that translates dialects, detects duplicates, and estimates project budgets:

Gemini REST API Integration: The system communicates directly with Google Gemini's endpoint using structured POST payloads, eliminating dependencies on third-party backend packages. 

Future<String?> _postToGemini(String prompt) async {
  final response = await http.post(
    Uri.parse('https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=$key'),
    headers: {'Content-Type': 'application/json'},
    body: json.encode({
      'contents': [{ 'parts': [{ 'text': prompt }] }]
    }),
  );
  if (response.statusCode == 200) {
    final data = json.decode(response.body);
    return data['candidates']?[0]?['content']?['parts']?[0]?['text']?.toString().trim();
  }
  return null;
}

11. Authentication & Security

JanSetu AI implements security gates for all user transactions and dashboard portals:

e-Pramaan SSO Gateway

Simulates authentication checks by requiring phone numbers and verifying OTP inputs to generate session tokens.

Secure Demo Sandbox

Includes pre-configured demo profiles, allowing evaluators and judges to log in immediately as a Citizen, MP, or Admin without setting up OTP details.

Cryptographic Token Verification

Every login issues a timed cryptographic token (e.g., `ePramaan-MP-MP-GUJ-SRT-01-1783526626175`), which is validated by Firestore security rules on every write request.

17. Challenges & Lessons

Bypassing Jekyll Blocks

Challenge: GitHub Pages ignores files starting with underscores, breaking Next.js assets.
Resolution: Created a `.nojekyll` file in the static directory to bypass Jekyll filtering.

Layout Overflows

Challenge: Multi-column headers caused layout overflows on small mobile viewports.
Resolution: Replaced rigid rows with responsive `Wrap` and `Flexible` grids.
